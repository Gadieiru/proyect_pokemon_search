# proyect_pokemon_search

Principalmente inicie con el tema de la base de datos para poder crear el API REST luego de esto segui con las connection querys, en este backend use unicamente JavaScript, nodeJS, MySQL y git para el manejo de repositorio.
Use mysql2 para el tema de conexiones con la base de datos, express en el tema de las dependencias y cors.
Para que el archivo principal (index.js) sea mas legible y entendible cree una carpeta config y dentro de esta un archivo connection, archivo el cual contiene la configuracion de conexion con la base de datos que nos proporcionara los datos provenientes de la base de datos.
Seguido a esto hice otra carpeta llamada routes el cual posee las rutas que manejara mi sitio web, en este caso estoy trabajando con pokemons, algo parecido a una pokedex, asi que los nombres concuerdan con lo que guarda la base de datos.
Finalmente importe todo al archivo index principal con app.use haciendolo asi mas legible y entendible para aquellos que quieran aprender facil y rapido.

Me enfrente a muchos problemas de logica, mas que todo en las peticiones get, que por momento es lo que quiero posea mi API, la llamada a los campos de las tablas en la base de datos se me dificultaban un poco pero esto no impidio que culminara el proyecto, aprendi de mis errores y aprendi mas formas de simplificar y ordenar un codigo, tambien todo el tiempo maneje las buenas practicas en todo momento asi que el codigo es muy entendible para todos, siendo sincero me gusto mucho hacer esto.
